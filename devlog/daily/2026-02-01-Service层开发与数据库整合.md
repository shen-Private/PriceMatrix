### 2. 修改 Controller 使用 Service

**HelloController.java 的改造：**

**改造前：**
```java
@GetMapping("/customer/search")
public Customer searchCustomer(@RequestParam Long id) {
    if (id == 1L) {
        return new Customer(1L, "张三", "zhang@example.com");
    } else if (id == 2L) {
        return new Customer(2L, "李四", "li@example.com");
    }
    return null;
}
```

**改造后：**
```java
private final CustomerService customerService;

public HelloController(CustomerService customerService) {
    this.customerService = customerService;  // Spring 自动注入
}

@GetMapping("/customer/search")
public Customer searchCustomer(@RequestParam Long id) {
    return customerService.findCustomerById(id);  // 委托给 Service
}
```

**理解的重点：**
- 职责分离：Controller 只负责接收/响应，Service 负责业务逻辑
- 建构子注入：Spring Boot 自动注入依赖

### 3. 把 Customer 改造成 JPA 实体

**model/Customer.java 的改造：**

**加上 JPA 注解：**
```java
@Entity  // 告诉 JPA 这是数据库实体
@Table(name = "customers")  // 表格名称
public class Customer {
    
    @Id  // 主键
    @GeneratedValue(strategy = GenerationType.IDENTITY)  // 自动递增
    private Long id;
    
    private String name;
    private String email;

    public Customer() {  // JPA 需要无参数建构子
    }
    
    // 其他建构子和 Getter...
}
```

**学到的注解：**
- `@Entity` - 标记为数据库实体
- `@Table(name = "customers")` - 指定表格名称
- `@Id` - 标记主键
- `@GeneratedValue` - 自动生成 ID

### 6. 修改 Service 使用 Repository

**CustomerService.java 最终版：**

```java
@Service
public class CustomerService {
    
    private final CustomerRepository customerRepository;
    
    public CustomerService(CustomerRepository customerRepository) {
        this.customerRepository = customerRepository;
    }
    
    public Customer findCustomerById(Long id) {
        Optional<Customer> customer = customerRepository.findById(id);
        return customer.orElse(null);  // 有值回传 Customer，没值回传 null
    }
    
    public List<Customer> findAllCustomers() {
        return customerRepository.findAll();
    }
}
```

**学到的概念：**
- `Optional<Customer>` - Java 8 的「可能有值也可能没值」容器
- `.orElse(null)` - 没值时回传 null


- Class = 文件（必须有 .java）

## 🧠 深入理解：Package 和 Import

### 重要发现

**Package ≠ 渲染开关，Package = 文件的地址**

**错误理解：**
> 「专案大时删掉第一行就不渲染某些功能」

**正确理解：**
```
package com.pricematrix.pricematrixdemo.service;
       ↓
这是文件的地址（邮寄地址）
就像：日本/和歌山县/海南市/XX町/XX番地
```

---

### Package 的作用

1. **告诉编译器：这个类别的完整名称**
   ```
   package com.pricematrix.service;
   public class CustomerService { }
   
   完整名称：com.pricematrix.service.CustomerService
   ```

2. **防止重名冲突**
   ```
   com.projectA.Customer
   com.projectB.Customer
   → 可以共存，因为地址不同
   ```

3. **让类加载器可以快速定位**
   ```
   import com.pricematrix.model.Customer;
   → JVM 知道去 com/pricematrix/model/Customer.class 找
   ```

---

### Import 的作用

**Import = 告诉 Java 去哪里找类别**

```java
package com.pricematrix.service;  // 我住在 service

import com.pricematrix.model.Customer;    // 我要找 model 的 Customer
import java.util.List;                    // 我要找 JDK 的 List
import org.springframework.stereotype.Service;  // 我要找 Spring 的 @Service
```

---

### 共用文件的正确做法

**场景：LoginService 想让多个 package 使用**

**❌ 错误做法（不可能）：**
```java
package com.pricematrix.packageA;  // ❌
package com.pricematrix.packageB;  // ❌ 一个文件不能有两个 package
```

**✅ 正确做法：**

```
1. LoginService 只有一个 package：
   package com.pricematrix.auth;

2. PackageA 想用 → import
   package com.pricematrix.packageA;
   import com.pricematrix.auth.LoginService;

3. PackageB 也想用 → 也 import
   package com.pricematrix.packageB;
   import com.pricematrix.auth.LoginService;
```

**比喻：**
- LoginService 住在一个家（auth）
- 大家都去他家找他
- 不是他同时住两个家

---

### 为什么需要明确声明 Package？

**看起来多余的原因：**
> 「文件已经在文件夹里了，为什么还要写 package？」

**实际原因：**

1. **编译器不看文件夹结构**
   - 编译器先读 package 声明
   - 再验证文件位置是否正确

2. **类加载器需要完整类名**
   - `import com.pricematrix.model.Customer`
   - JVM 转成路径：`com/pricematrix/model/Customer.class`
   - 快速加载

3. **防止文件被移动时悄悄出错**
   - 如果自动判断，移动文件后 package 会悄悄改变
   - 明确声明后，位置错误会立刻报错

4. **支援多源码目录**
   ```
   src/main/java/com/pricematrix/Customer.java
   src/test/java/com/pricematrix/CustomerTest.java
   ```
   两个都在相同路径，靠 package 声明区分

### Hibernate 建表过程

```sql
Hibernate: drop table if exists customers cascade 
Hibernate: create table customers (
    id bigint generated by default as identity, 
    email varchar(255), 
    name varchar(255), 
    primary key (id)
```

**理解：**
- `ddl-auto=create-drop` 会在启动时删除并重建表格
- `id bigint generated by default as identity` - 自动递增主键

### 数据插入过程

```sql
Hibernate: insert into customers (email,name,id) values (?,?,default)
Hibernate: insert into customers (email,name,id) values (?,?,default)

```

**理解：**
- `?` 是预编译语句的参数占位符（防 SQL 注入）
- `id` 用 `default` 表示使用自动递增

## 🎯 学到的核心概念

### 1. 职责分离（Separation of Concerns）

```
Controller: 接收 HTTP → 回传 JSON
Service:    业务逻辑
Repository: 数据存取（SQL 操作）
Database:   储存数据
```

每一层只做自己的事，不越界。

---

### 2. 依赖注入（Dependency Injection）

**传统写法（不好）：**
```java
public class CustomerService {
    private CustomerRepository repo = new CustomerRepository();  // 自己 new
}
```

**Spring Boot 写法（好）：**
```java
public class CustomerService {
    private final CustomerRepository repo;
    
    public CustomerService(CustomerRepository repo) {
        this.repo = repo;  // Spring 注入
    }
}
```

**好处：**
- 解耦：不需要知道怎么建立 Repository
- 可测试：可以注入假的 Repository
- 可替换：想换实现只要改设定

---

### 3. Spring Data JPA 的魔法

**你写的：**
```java
public interface CustomerRepository extends JpaRepository<Customer, Long> {
    // 空的！
}
```

**Spring 自动提供：**
```java
findById(Long id)
findAll()
save(Customer customer)
deleteById(Long id)
count()
// ... 还有很多
```

**原理：动态代理（Dynamic Proxy）**

---

### 4. Hibernate ORM

**ORM = Object-Relational Mapping（物件关联映射）**

**作用：**
```
Java 物件 ←→ 数据库表格

Customer 类别 ←→ customers 表格
id 字段      ←→ id 栏位
name 字段    ←→ name 栏位
```

**好处：**
- 不用写 SQL
- 用物件的方式操作数据库
- 自动转换

---

### 5. JSON 序列化

**流程：**
```
Controller 回传 List<Customer>
    ↓
Spring MVC 拦截
    ↓
用 Jackson 转换
    ↓ (用 Reflection 找 Getter → 取值)
回传 JSON 字串
```

**为什么需要 Getter？**
- Jackson 靠 Getter 取值
- 没有 Getter 就无法序列化

---

### 6. Optional<T>

**传统写法（容易出错）：**
```java
Customer customer = repository.findById(id);
if (customer != null) {
    return customer;
}
return null;
```

**Optional 写法（更安全）：**
```java
Optional<Customer> customer = repository.findById(id);
return customer.orElse(null);
```

## 🤔 困惑与解答

### Q1: 为什么 Repository 是 interface 却能直接用？

**A:** Spring Data JPA 在启动时用「动态代理」自动产生实现类别。

### Q2: 为什么需要无参数建构子？

**A:** JPA/Hibernate 用 Reflection 建立物件时，需要先 `new Customer()`，再用 Reflection 填数据。

### Q3: Package 不能删吗？

**A:** 不能！Package 是文件的地址，删掉就找不到这个类别了。

### Q4: 为什么一个文件不能有两个 package？

**A:** 因为 package 就是地址，一个人不能同时住两个地方。

### Q5: 怎么让多个 package 共用一个类别？

**A:** 类别只有一个 package，其他 package 用 import 去找它。
