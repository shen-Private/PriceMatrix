💡 重要概念理解
Node.js vs React 的關係
Node.js ≠ 前端技術
Node.js = JavaScript 執行環境

在 PriceMatrix 專案中：
├─ Node.js：開發工具（npm 套件管理）
├─ React：前端框架（瀏覽器執行）
└─ Spring Boot：後端框架（JVM 執行）
npm 的角色
npm = 套件管理器（不是套件庫本身）
    = 程式碼的「應用程式商店」

類比：
npm          = 7-11 便利商店（通路）
React        = 可口可樂（商品）
Next.js      = 星巴克咖啡（商品）
React vs Next.js
React    = 基礎框架
Next.js  = React 的加強版（包含 React + 路由 + SSR）

選擇 React 的原因：
├─ PriceMatrix 是內部系統（不需要 SEO）
├─ 學習曲線較平緩
└─ 概念更單純，適合初學
React 的核心優勢
不是「能做什麼」，而是「怎麼組織 code」
概念WordPress/jQueryReact元件重用include（單純貼 code）真正的元件（可傳參數、有狀態）狀態管理全域變數或手動追蹤useState 集中管理更新畫面手動操作 DOM自動重新渲染事件綁定.on() 手動綁定JSX 直接綁定

🎓 今日學會的技術
1. State（狀態管理）
jsxconst [searchText, setSearchText] = useState('');
      ^^^^^^^^^^  ^^^^^^^^^^^^^     ^^^^^^^^^^^
      目前的值     改變值的方法      初始值
WordPress 類比：
php// WordPress
global $search_query;

// React
const [searchText, setSearchText] = useState('');
核心差異：

WordPress：手動追蹤狀態
React：改 state → 畫面自動更新


2. 事件處理
jsx// 輸入框
<input 
    value={searchText}
    onChange={(e) => setSearchText(e.target.value)}
/>

// 按鈕
<button onClick={handleSearch}>搜尋</button>
WordPress/jQuery 做法：
javascript$('#input').on('input', function() {
    var value = $(this).val();
    $('#output').text(value);  // 手動更新 DOM
});
React 做法：
jsx// 只要改 state，React 自動更新畫面
setSearchText(value);

3. JSX 語法
JSX = HTML + JavaScript 混合
jsx<p>你輸入的是：{searchText}</p>
             ^^^^^^^^^^^^
             JavaScript 表達式
WordPress 類比：
php<p>你輸入的是：<?php echo $search_text; ?></p>
```

**就是把 PHP 的 `<?php echo ?>` 改成 `{}`**

### 4. Props（父子元件通訊）

**完整流程：**
```
App.js（父）
│
├─ 1. 定義函數：handleCustomerSearch
│       ↓
├─ 2. 傳給子元件：
│      <CustomerSearch onSearch={handleCustomerSearch} />
│       ↓
│   CustomerSearch（子）
│   │
│   ├─ 3. 接收：function CustomerSearch({ onSearch })
│   │       ↓
│   └─ 4. 呼叫：onSearch('沈')
│       ↓
└─ 5. 執行：handleCustomerSearch('沈')
        ↓
    setSelectedCustomer('沈')
        ↓
    畫面更新
```

**WordPress 很難做到（要用全域變數或 AJAX）**  
**React 超簡單！**

📊 CustomerSearch.js 完整程式碼
jsximport { useState } from 'react';

function CustomerSearch({ onSearch }) {
  const [searchText, setSearchText] = useState('');

  const handleSearch = () => {
    onSearch(searchText);  // 把資料傳給父元件
  };

  return (
    <div>
      <h2>客戶搜尋</h2>
      <input 
        type="text" 
        placeholder="輸入客戶名稱..."
        value={searchText}
        onChange={(e) => setSearchText(e.target.value)}
      />
      <button onClick={handleSearch}>搜尋</button>
      <p>你輸入的是：{searchText}</p>
    </div>
  );
}

export default CustomerSearch;

📊 App.js 完整程式碼
jsximport './App.css';
import CustomerSearch from './CustomerSearch';
import { useState } from 'react';

function App() {
  const [selectedCustomer, setSelectedCustomer] = useState('');

  const handleCustomerSearch = (customerName) => {
    setSelectedCustomer(customerName);
  };

  return (
    <div className="App">
      <CustomerSearch onSearch={handleCustomerSearch} />
      <hr />
      <h3>App.js 收到的客戶名稱：{selectedCustomer}</h3>
    </div>
  );
}

export default App;
```

## 💭 學習心得

### 前端比後端舒服的原因

**1. 本來就在這個生態圈**
```
WordPress 開發：
├─ HTML/CSS 精通 ✓
├─ 理解 DOM 結構 ✓
└─ 視覺化思維 ✓

React 開發：
└─ 用的是相同的底層知識
└─ 只是換個方式組織 code
```

**2. 即時回饋**
```
後端（Java）：
└─ 寫 code → 編譯 → 測試 → 看結果（過程較長）

前端（React）：
└─ 寫 code → 儲存 → 立刻看到畫面變化（成就感快）
```

**3. 視覺化更直觀**
```
後端邏輯：抽象的（要想像資料流）
前端元件：具體的（直接看到按鈕、輸入框）
```
