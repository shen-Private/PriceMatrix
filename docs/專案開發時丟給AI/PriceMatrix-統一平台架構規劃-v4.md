## 3. 客戶管理頁面設計

### 3.1 多事務所處理（parent_id 方案）

同一公司旗下多個事務所各自是一筆 customer，透過 parent_id 指向上層公司。

| 情境 | 做法 |
|------|------|
| 獨立客戶 | parent_id = NULL |
| 山田介護（總公司） | parent_id = NULL（上層） |
| 山田介護 渋谷事務所 | parent_id = 山田介護 id |
| 山田介護 新宿事務所 | parent_id = 山田介護 id |

### 3.2 展開行 Tab 結構

點擊客戶行展開，顯示四個 Tab：

| Tab | 內容 |
|-----|------|
| 時間軸 | 報價單 + 訂單 + 聯絡紀錄合併，依時間倒序排列 |
| 報價單 | 該客戶的報價單列表（前端過濾） |
| 訂單 | 該客戶的訂單列表（前端過濾） |
| 聯絡紀錄 | 聯絡紀錄時間軸 + 右側新增表單 |

### 3.3 客戶擔當分配（待做）

`pricing_customers` 加 `assigned_to` 欄位，記錄負責業務帳號。
業務離職時只需改欄位，不需 copy 資料，歷史紀錄完整保留。

### 3.4 成交流程（方案B：手動確認，待做）

業務按「成交」→ 跳出表單補齊正式客戶資料 → 建立後自動關聯 `prospect.customer_id`。

## 4. 日報填寫頁面設計

### 4.1 填寫欄位

| 欄位 | 類型 | 說明 |
|------|------|------|
| 互動方式 | 下拉選單 | 拜訪 / 電話 / 郵件 / 報價 / 其他 |
| 日期時間 | datetime-local | 聯絡時間 |
| 結果 | 下拉選單 | 待跟進 / 正面 / 負面 |
| 內容 | 自由文字 | 當天發生的事 |
| 下次行動 | 自由文字 | AI 分析用 |

### 4.2 填寫位置

客戶管理頁面展開行右側，不另開獨立頁面。
