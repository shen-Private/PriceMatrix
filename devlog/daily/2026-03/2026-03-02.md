## 踩到的坑

### ZXing callback 殘留問題
**症狀：** 掃完條碼後，再次開相機會自動填入上次的條碼。

**原因：** `decodeFromVideoDevice` 回傳一個 `controls` 物件。
只停 video stream 不夠，ZXing 的 decode loop 還在跑，舊 callback 繼續觸發。

**解法：** 儲存 `controls`，`stopScan` 時呼叫 `controls.stop()`：

```tsx
const controlsRef = useRef<any>(null);

// startScan 裡
const controls = await reader.decodeFromVideoDevice(...);
controlsRef.current = controls;

// stopScan 裡
if (controlsRef.current) {
    controlsRef.current.stop(); // ← 這行是關鍵
    controlsRef.current = null;
}
```
