## 概念學習

### React 設計理念

**核心：UI 是狀態的函數 `UI = f(state)`**

- 不去操作 DOM，只管理狀態，畫面自動跟著狀態走
- React 偵測不到直接修改物件，必須透過 `setState` 才會觸發重新渲染

**渲染流程：**
```
狀態改變（setState）
    ↓
React 重新執行元件函式
    ↓
算出新的虛擬 DOM
    ↓
跟舊的虛擬 DOM 比較（diff）
    ↓
只更新真正有差異的地方
```

**對比傳統 jQuery：**
```javascript
// jQuery：你親自去找元素、親自改它
$('#count').text(newValue)

// React：你只改狀態，畫面自己更新
setCount(newValue)
```

### `...prev` 展開運算子的原因

`useState` 的修改函式是「整個替換」，不是「局部修改」。

```typescript
// shipments 是物件，存多筆訂單的出貨資料
// 只想更新某一筆，但 setShipments 是整個替換
// 所以要先展開舊的，再蓋掉想改的那筆

setShipments(prev => ({
  ...prev,             // 保留其他所有訂單的資料
  [orderId]: res.data  // 只蓋掉這一筆
}))
```

### TypeScript 設計思路

**存在理由：在程式跑起來之前，先抓到錯誤**

```typescript
// JS：跑起來才爆炸
function getPrice(product) {
  return product.price * 1.1
}
getPrice("printer")

// TS：寫的時候 VSCode 就畫紅線
function getPrice(product: Product) {
  return product.price * 1.1
}
getPrice("printer")  // ❌ 編譯錯誤
```

**三個主要使用場景：**
```typescript
// 1. 變數
const count: number = 0

// 2. 函式參數／回傳值
function add(a: number, b: number): number {
  return a + b
}

// 3. 物件形狀（interface）
interface Customer {
  id: number
  name: string
  active: boolean
}
```

### 前後端型別分工

```
後端（Java）   → 強制型別，編譯時檢查
前端（TS）     → 強制型別，編譯時檢查
中間（JSON）   → 純文字，沒人管
```

- TS interface 只存在於前端編譯階段，跑起來就消失
- 兩邊各自管自己，中間靠「約定」維持一致
- 後端偷偷改格式，TS 不知道——只會在瀏覽器跑起來才發現

**大型專案的解法：OpenAPI / Swagger**
- 從 Java annotation 自動掃描，產生 API 文件
- 工具再從文件自動產生前端 interface
- 後端改欄位 → 重新產生一次 → 前端自動同步，不用人工對齊
- PriceMatrix 規模不需要，手動維持兩邊一致即可

### TS interface 的順序與數量規則

- **順序：不管**（物件欄位順序不影響型別檢查）
- **數量：嚴格管**（少一個或多一個都報錯）

```typescript
interface Customer {
  id: number
  name: string
}

const c: Customer = { name: "山田", id: 1 }  // ✅ 順序不同沒問題
const c: Customer = { id: 1 }               // ❌ 少 name
const c: Customer = { id: 1, name: "山田", extra: "?" }  // ❌ 多 extra
```

**`?` 可選欄位：**
```typescript
interface Customer {
  id: number
  name: string
  phone?: string  // 有沒有都可以
}
```

**PriceMatrix 不大量使用 `?` 的理由：**
- 業務上必填的欄位（名稱、代號）本來就不該是可選
- `?` 會讓每次使用都需要判斷 `undefined`，程式碼變亂
- 原則：**有沒有都不影響核心功能的欄位才用 `?`，讓 TS 強制執行業務規則**
