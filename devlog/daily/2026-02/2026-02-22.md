## 踩過的坑

**inventory 資料夾放錯位置**
用 IntelliJ 拖移時，inventory 跑到 `java\inventory\` 而不是 `java\com\pricematrix\pricematrix\inventory\`。用 CMD 的 `move` 指令修正。

**Lombok 沒有加進 pom.xml**
inventory 的新檔案全部用了 `@Data`，但 pom.xml 裡沒有 Lombok dependency，導致 47 個錯誤。加入後解決。

**Lazy Loading 導致 JSON 序列化失敗**
`InventoryItem` 的 `product` 欄位用 `FetchType.LAZY`，Jackson 轉 JSON 時抓不到資料，報 `ByteBuddyInterceptor` 錯誤。在 `product` 欄位加上 `@JsonIgnoreProperties({"hibernateLazyInitializer", "handler"})` 解決。

**DataSeeder import 路徑過時**
pricing 模組移動後，`DataSeeder.java` 裡的 `import com.pricematrix.pricematrix.repository.*` 需要改為 `import com.pricematrix.pricematrix.pricing.repository.*`。

## 學到的概念

**Lombok**
Java 的工具套件，`@Data` 標注自動產生所有 getter/setter，減少樣板代碼。需要在 pom.xml 加 dependency 才能使用。

**Lazy Loading vs Eager Loading**
`FetchType.LAZY` 是「用到才載入」，效能好但 JSON 序列化時可能抓不到資料。解法是加 `@JsonIgnoreProperties` 讓 Jackson 忽略 Hibernate 的代理物件。

**CMD move 指令**
`move "來源路徑" "目標路徑"` 可以移動整個資料夾，但 IntelliJ 不會自動更新 package 宣告，需要手動修正 import。
