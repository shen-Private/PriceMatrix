## 釐清的概念

### 資料庫三層概念
- 軟體（PostgreSQL / MySQL）、伺服器（本地 / Neon / Render）、資料庫實例（庫）三層分開理解
- 模組間用 table 前綴區分（`pricing_` / `inventory_` / `sales_`），同一個庫
- 微服務才會拆成多個庫，目前規模不需要

### psql 指令 vs SQL 語法
- `\d` / `\dt` / `\l` 是 psql 專用指令
- GUI 或其他工具要用 `information_schema` 查表結構

### Entity 耦合問題（技術債確認）
- `SalesQuoteItem` 直接引用 `Product`（pricing 模組），跨模組耦合
- `SalesQuote` 直接引用 `Customer`，同樣問題
- 正確做法：改為 `productId` + 快照欄位（`productName` / `basePrice`）
- **暫不處理，列入技術債**

### Spring Boot 語法理解
- `@ManyToOne` / `@OneToMany`：表達 table 之間的關聯，Hibernate 自動 JOIN
- `@Column(nullable = false)`：對應 DB 的 NOT NULL
- `public class` vs `private` 欄位：class 是對外可見，欄位透過 getter/setter 存取
- `@Data`（Lombok）：自動產生所有 getter / setter
- `new`：用 class 建立物件實體
- `BigDecimal`：金額專用型別，避免 `double` 的精度問題
- class 同時是型別定義和設計圖

### 權限設計
- JWT 存靜態權限（role），動態權限（誰能改哪筆資料）需查 DB
- 頻繁查 DB 不等於高安全，是舊系統 Session 時代的遺留設計
- 真正高安全：JWT 短效期 + Refresh Token + 敏感操作才重新驗證
