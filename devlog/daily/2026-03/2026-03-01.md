## 學到的概念

### Props：父元件傳資料給子元件

props 是「父元件傳進來的所有東西」的統稱，React 自動打包成物件傳入。

```jsx
// 父元件傳入
<UserCard role="admin" name="王小明" />

// 子元件接收（物件解構）
function UserCard({ name, role }) { ... }

// 等同於
function UserCard(props) {
  props.name  // "王小明"
  props.role  // "admin"
}
```

**重點：**
- `{ name, role }` 是物件解構，不是陣列
- 資料只能從父傳子（單向），子無法傳回父
- 字串用引號傳：`name="王小明"`，變數用大括號傳：`name={userName}`

**元件命名規則：**
- 小寫開頭 `<button>` → HTML 原生標籤
- 大寫開頭 `<UserCard>` → 自訂 React 元件（React 規定）

---

### 為什麼需要子元件

共用結構抽出來，變動的從外面給。

```jsx
// ❌ 沒有子元件：重複寫100次
function Practice() {
  return (
    <div>
      <div><p>商品A</p><p>$100</p></div>
      <div><p>商品B</p><p>$200</p></div>
      // ...
    </div>
  );
}

// ✅ 有子元件：結構只寫一次
function ProductCard({ name, price }) {
  return (
    <div>
      <p>{name}</p>
      <p>{price}</p>
    </div>
  );
}

function Practice() {
  return (
    <div>
      <ProductCard name="商品A" price="$100" />
      <ProductCard name="商品B" price="$200" />
    </div>
  );
}
```

---

### Fragment：避免多餘的 HTML 標籤

return 必須有一個根元素，但不想多一個 div 的話用 Fragment：

```jsx
return (
  <>
    <p>第一行</p>
    <p>第二行</p>
  </>
);
```

`<>` 和 `</>` 不會渲染成任何 HTML 標籤。

### 條件渲染 + style

判斷邏輯先算好，再放進 style：

```jsx
function UserCard({ name, role }) {
  let bgColor;
  if (role === "admin") {
    bgColor = "red";
  } else if (role === "user") {
    bgColor = "blue";
  }

  return (
    <p style={{ backgroundColor: bgColor }}>{role} {name}</p>
  );
}
```

**三元運算子寫法（簡短版）：**
```jsx
let textColor = price > 50 ? "red" : "";
```

---

### map 資料渲染

```jsx
const products = [
  { id: 1, name: "蘋果", price: 30 },
  { id: 2, name: "香蕉", price: 15 },
  { id: 3, name: "西瓜", price: 80 },
];

return (
  <div>
    {products.map(product => {
      let textColor = product.price > 50 ? "red" : "";

      return (
        <p key={product.id} style={{ color: textColor }}>
          {product.name} - ${product.price}
        </p>
      );
    })}
  </div>
);
```

**重點：**
- `product` 是自己取的名字，慣例用單數
- `.name` / `.price` 是資料的欄位名稱
- `key` 是給 React 內部識別用的，必須唯一，慣例用 id
- 條件判斷要放在 map 裡面，每筆資料各自判斷
- map 裡的 return 是箭頭函數的 return，不是元件的 return

---

### return 兩層的解釋

```jsx
// 外層：元件函數的 return
return (
  <div>
    {products.map(product => {

      // 內層：map 箭頭函數的 return
      return (
        <p key={product.id}>{product.name}</p>
      );
    })}
  </div>
);
```

兩個不同的函數，各自有自己的 return，沒有衝突。

### 固定值 vs 動態資料

練習階段手動寫死：
```jsx
<UserCard role="admin" name="王小明" />
```

正式接後端：
```jsx
useEffect(() => {
  axios.get('/api/users').then(response => setUsers(response.data));
}, []);

return (
  <div>
    {users.map(user => (
      <UserCard key={user.id} role={user.role} name={user.name} />
    ))}
  </div>
);
```
