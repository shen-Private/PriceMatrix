## 學到的概念

### useState

React 的 Hook，讓 component 記住資料，改了就自動重新渲染。

```tsx
const [items, setItems] = useState<ItemWithStock[]>([]);
// items    → 拿來讀的（現在的值）
// setItems → 拿來改的（改了觸發重新渲染）
// []       → 初始值
```

- React 在背後維護記憶體格子，每個 `useState` 對應一個格子
- `items` 在同一次渲染裡是不變的 `const`，每次渲染 React 從格子取最新值給它
- 改值只能用 `setItems`，不能直接 `items = xxx`
- `setItems` 呼叫後不是立刻生效，是下次渲染才生效

### useEffect

控制「在什麼時機執行某件事」的 Hook。

```tsx
useEffect(() => {
  loadItems(); // 要執行的事
}, []);        // 依賴陣列：[] = 只在第一次渲染執行
```

依賴陣列的三種寫法：
- `[]` → 只執行一次（頁面載入時）
- `[filterType]` → filterType 改變時執行
- 沒有第二個參數 → 每次渲染都執行（幾乎不用，容易無限迴圈）

跟 `useState` 的標配組合：宣告空資料 → 頁面出現 → useEffect 觸發 → 打 API → 資料回來 → setState → 畫面更新

### useRef

存東西，但改了不觸發重新渲染。適合存 timer ID 等不需要更新畫面的東西。

```tsx
const toastTimer = useRef<ReturnType<typeof setTimeout> | null>(null);
```

### axios

HTTP 請求的工具套件，比原生 `fetch` 更好用。
自動處理 JSON 轉換，`res.data` 直接就是後端回傳的資料本體。

### component

回傳畫面的函數，可以重複使用。跟 Figma 的 component 概念相同。
目前 `InventoryPanel` 是整頁級別，之後會拆分成更小的 component。

### TypeScript interface

定義物件的「形狀」，對應後端的 Entity。
不是執行時的檢查，是開發時讓編譯器幫你抓錯的工具。

```tsx
interface InventoryItem {
  id: number;
  unit: string;
  safetyStock: number | null;  // 聯合型別：數字或 null
  isActive: boolean;
}
```

### TypeScript 型別斷言（as）

告訴編譯器「我知道這個型別是什麼」。

```tsx
Object.keys(STOCK_TYPE_LABEL) as StockType[]
// string[] → StockType[]
```

### React key prop

用 `map` 產生多個元素時，每個元素必須有唯一的 `key`，讓 React 知道哪個元素對應哪筆資料，只更新需要更新的部分。

### DOM 與虛擬 DOM

- **真實 DOM**：瀏覽器對 HTML 的樹狀記憶體結構
- **操作真實 DOM**：直接用 JS 修改這個結構（`document.getElementById` 等）
- **虛擬 DOM**：React 在記憶體裡維護的 DOM 副本
- React 先在虛擬 DOM 比對差異，只把有差異的部分更新到真實 DOM，減少瀏覽器工作量

### React vs AJAX vs 傳統網頁

```
傳統網頁  → 整頁重新載入
AJAX     → 不重新整理就能拿資料，但畫面更新要手動操作 DOM
React    → 拿到資料後，畫面更新全部自動處理
```

AJAX 解決「怎麼拿資料」，React 解決「拿到資料後怎麼更新畫面」。

## 架構理解

**React 對 PriceMatrix 的實際價值**
效能（虛擬 DOM）不是主要理由，因為同時在線人數少、操作頻率低。
真正的價值是：component 化（好維護、好重用）、state 管理（資料和畫面自動同步）、架構清晰。

**JS / TS / React 的關係**
- JS：語言基礎（`Object.keys`、`map`、陣列操作）
- TS：JS 加上型別系統（`interface`、`as`、`Record`）
- React：框架（`useState`、`useEffect`、`component`、JSX）
