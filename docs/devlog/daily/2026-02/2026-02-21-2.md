## 踩過的坑

**psql 連的是 Render 而非本地**
執行 `psql` 後以為連的是本地，實際上連的是 Render。本地 DB 名稱是 `pricematrix`，Render 是 `pricematrix_db`。本地連線指令：
```bash
psql -U postgres -d pricematrix
```

**500 錯誤：column is_active does not exist**
原因是本地 DB 缺欄位。對本地 `pricematrix` 執行 ALTER TABLE 後解決。

---

## 架構決策

**新增折扣 = 選現有商品，不允許手打**
業務不需要在新增折扣時建立新商品，商品管理是獨立的功能。

**商品軟刪除**
商品不直接刪除，改用 `is_active = false` 標記停用。停用後不出現在新增折扣的下拉選單，但歷史折扣紀錄不受影響。
