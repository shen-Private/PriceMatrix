
## 🧠 今日學到的概念（2026-02-15）

### JavaScript / React 基礎

#### `return` 的真正意思
- **不是**啟動函數
- 是把結果**交回給呼叫者**
- 誰呼叫我，就回傳給誰
- JS 和 Java 的 return 概念完全一樣

#### `const` 的意思
- **不是**呼叫函數
- 是**宣告變數**
- `const result = add(1, 2)` 拆開看：
  - `add(1, 2)` → 呼叫函數（啟動）
  - `return` → 函數把結果丟出來
  - `const result =` → 把結果存起來

#### `onChange` 事件
- 每次輸入框有變化，自動觸發
- `e.target.value` = 輸入框裡的值
- React 哲學：資料跟畫面永遠同步，不用手動抓值

#### 三元運算子 `? :`
- 等於 if / else
- 可以寫在 JSX 裡面（if/else 不行）
- 巢狀三元 = else if
```javascript
條件A ? 結果A
  : 條件B ? 結果B
  : 結果C
```

#### `Array.map()`
- JS 原生方法，React 沒有發明新東西
- 把陣列每個元素，轉換成另一個東西
- React 裡用來把資料轉換成 JSX 標籤

### React 概念

#### 編輯模式 / 顯示模式切換
- 用一個狀態（`editingId`）控制「哪一格在編輯」
- `editingId === discount.id` → 顯示輸入框
- `editingId !== discount.id` → 顯示文字
- `onBlur` → 點別處自動取消編輯

#### JSX 是什麼
- JSX = JavaScript + XML
- 長得像 HTML，但規則是 JavaScript 的
- 背後會被轉換成 `React.createElement()`

| HTML 寫法 | JSX 寫法 |
|-----------|----------|
| `class="box"` | `className="box"` |
| `style="color:red"` | `style={{ color: 'red' }}` |
| `<br>` | `<br />` |

### 延伸知識

#### XML vs HTML 的關係

| | 用途 | 標籤 |
|---|---|---|
| XML | 儲存 / 傳遞資料 | 自訂 |
| HTML | 顯示網頁 | 固定 |
| JSX | React 的語法 | HTML + 自訂元件 |

- XML 制定了「用標籤包資料」的規則
- HTML 借用這個概念，但標籤固定，專給瀏覽器看
- JSX 也借用這個概念，加入自訂元件

### Java 概念

#### getter / setter
- `get` → 取出物件的屬性值
- `set` → 設定物件的屬性值
- 標準寫法：`existing.setDiscountRatio(updatedDiscount.getDiscountRatio())`

#### `Optional` 與 `orElseThrow`
- `findById()` 回傳 `Optional<Discount>`
- Optional = 「可能有、可能沒有」的容器
- `.orElseThrow()` → 有就拿出來，沒有就丟錯誤

#### `save()` 的邏輯
- 有 id → UPDATE（更新）
- 沒有 id → INSERT（新增）

時機：Spring Boot 啟動時自動執行
特性：H2 是記憶體資料庫，重啟後資料重置

#### 3. CORS 是什麼
問題：前端（:3000）呼叫後端（:8080）= 跨域請求
瀏覽器預設會擋掉
解法：@CrossOrigin(origins = "http://localhost:3000")
意思：告訴瀏覽器「這個來源是被允許的」

#### 4. Interface vs Class（Java）
Interface = 規格書（定義能做什麼）
Class = 實作（真正執行）

CustomerRepository：
你只寫 Interface（規格）
Spring JPA 自動幫你實作（findAll, save, delete...
）
#### 5. Port以及CORS
Port = 同一台電腦裡不同程式的門牌號碼
CORS = 解除瀏覽器跨域限制，不是安全機制
真正的安全防線是 JWT Token + HTTPS
開發階段 HTTP 沒問題，上線前再處理
