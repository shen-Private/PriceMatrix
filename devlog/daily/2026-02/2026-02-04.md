## 🧠 核心概念已理解

### 前端相關（新增）

#### 1. Node.js 的角色
```
Node.js ≠ 前端技術
Node.js = JavaScript 執行環境

在開發 React 時：
├─ npm：套件管理器
├─ React Dev Server：本地開發伺服器
└─ Webpack/Babel：打包和編譯工具

Node.js 也能做後端（但我們用 Spring Boot）
```

#### 2. npm 的本質
```
npm = 套件管理器（不是套件庫）

類比：
npm      = 7-11 便利商店（通路）
React    = 可口可樂（商品）
axios    = 礦泉水（商品）
```

#### 3. React 的核心優勢
```
不是「能做什麼」
而是「怎麼組織 code」

vs WordPress/jQuery：
├─ WordPress：include（單純貼 code）
├─ React：真正的元件（可傳參數、有狀態）

vs 手動操作 DOM：
├─ jQuery：手動更新畫面
├─ React：改 state → 自動更新




### 2. 编译型 vs 解释型

**编译型 (Java):**
```
写 code → 编译成 .class → 执行
优点: 执行快
缺点: 改了要重新编译
```

**解释型 (PHP):**
```
写 code → 直接执行
优点: 改了马上生效
缺点: 执行较慢
```

**Hot Reload 最佳平衡:**
```
Java + Spring Boot DevTools
= 编译型的效能 + 解释型的方便性
```

---

4. State（狀態管理）
jsxconst [searchText, setSearchText] = useState('');
      ^^^^^^^^^^  ^^^^^^^^^^^^^     ^^^^^^^^^^^
      目前的值     改變值的方法      初始值
WordPress 類比：
php// WordPress：全域變數
global $search_query;

// React：useState
const [searchText, setSearchText] = useState('');
5. 事件處理
jsx// 輸入時
<input onChange={(e) => setSearchText(e.target.value)} />

// 點擊時
<button onClick={handleSearch}>搜尋</button>
vs jQuery：
javascript// jQuery：手動綁定 + 手動更新 DOM
$('#input').on('input', function() {
    var value = $(this).val();
    $('#output').text(value);
});

// React：只要改 state，畫面自動更新
setSearchText(value);
6. JSX 語法
jsx<p>你輸入的是：{searchText}</p>
             ^^^^^^^^^^^^
             JavaScript 表達式
vs PHP：
php<p>你輸入的是：<?php echo $search_text; ?></p>
```

**就是把 `<?php echo ?>` 改成 `{}`**

#### 7. Props（父子元件通訊）
```
父元件（App.js）
    ↓ 傳函數
子元件（CustomerSearch）
    ↓ 呼叫函數傳資料
父元件收到資料
```

**WordPress 很難做到（要用全域變數或 AJAX）**  
**React 超簡單！**

### 後端相關（之前學的）

#### 1. Java vs JDK vs Spring Boot
```
Java        → PHP 語言
JDK         → PHP 安裝包
Spring Boot → WordPress
```

#### 2. 四層架構
```
前端 (React + TS)
    ↓
Controller (接收/響應層)
    ↓
Service (業務邏輯層)
    ↓
Repository (持久化層)
    ↓
資料庫 (H2 → 未來換 MySQL)
3. 依賴注入（DI）
java// Spring 自動注入
public CustomerService(CustomerRepository repo) {
    this.repo = repo;  // Spring 幫你 new
}
4. Spring Data JPA 的魔法
java// 你只寫 interface
public interface CustomerRepository extends JpaRepository<Customer, Long> {
}

// Spring 自動提供：findById, findAll, save, delete...
```

#### 5. Package 和 Import（重要！）
```
package = 檔案的地址（不是渲染開關）
import  = 告訴 Java 去哪找類別

一個檔案只能有一個 package
其他人想用 → import 這個 package
```
