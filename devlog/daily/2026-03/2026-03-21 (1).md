## 技術債處理

### SalesQuoteItem 耦合修正 ✅

**問題：** `SalesQuoteItem` 直接引用 `pricing` 模組的 `Product` Entity（跨模組耦合）

**修正內容：**

#### DB
```sql
ALTER TABLE sales_quote_item
  ADD COLUMN product_name VARCHAR(255),
  ADD COLUMN product_code VARCHAR(100),
  ADD COLUMN base_price NUMERIC(38,2);
```

#### SalesQuoteItem.java
- 移除 `@ManyToOne Product product`
- 改為 `Long productId`（純 ID）
- 新增快照欄位：`productName` / `productCode` / `basePrice`
- 移除 `import Product`

#### SalesQuoteService.java
- `createQuote` / `createRevision` / `updateQuote` 三處改為寫入快照欄位
- `item.setProduct(product)` → `item.setProductId / setProductName / setBasePrice`
- 注意：`Product.getName()`（不是 `getProductName()`）

**學習重點：**
- 快照（Snapshot）= 建立時複製當下的值，之後與原始資料脫鉤
- 報價單是財務文件，商品改名/改價/下架不應影響歷史記錄
- `@ManyToOne` 存的是物件引用，Hibernate 取資料時自動 JOIN 查當下值

## 概念學習（學習日）

### HTTP / 網路基礎

- **後端 vs 前端學習感：** 後端更像「學框架規則」（annotation 描述意圖，框架做底層）；前端更像「學程式思維」（狀態與資料流自己寫）
- **Spring Boot 幫你處理的三件事：**
  - HTTP 解析：把瀏覽器送來的文字解析成方法/路徑/參數
  - 物件生命週期：`@Service` 標記後 Spring 負責 new / 注入，不用手動管理
  - 資料庫連線池：維護一批連線，查詢時自動借用歸還，不用每次重新建立
- **HTTP vs HTTPS：** HTTP 是規則（明文），HTTPS 是加密版。本地開發用 HTTP 因為沒有竊聽風險，部署後平台自動處理 HTTPS
- **HTTPS 保護範圍：** 只保護瀏覽器→伺服器的傳輸途中；後端→DB 有另外的 TLS（Neon 的 `sslmode=require`）
- **Cookie：** 瀏覽器自動管理的 key=value 儲存，`HttpOnly` 讓 JS 無法讀取但瀏覽器發請求時底層自動附上（不經過 JS）
- **Token（HTTP）：** 登入後後端產生的加密通行證（JWT），跟 AI 的 token（文字計量單位）完全不同

### TypeScript 基礎

- **`interface`：** 定義物件形狀，欄位名稱固定，多一個少一個都報錯
- **`type`：** 與 interface 類似，也可定義聯合型別 `type Role = 'admin' | 'sales'`
- **`Record<K, V>`：** TS 內建工具型別，key/value 型別固定但欄位名稱不限
- **`|`（聯合型別）：** 「或」的意思，限制值只能是某幾個選項
- **`React.FC`：** 標注函式是 React 元件，現代專案可省略
- **閱讀技巧：** 看不懂 TS 時，先在腦袋裡把型別標注（`: number` / `<T>`）拿掉，剩下就是純 JS 邏輯

### React 基礎

- **`interface` vs `useState` 的分工：** interface = 設計圖（給 TS 看，執行時不存在）；useState = 實際建立儲存資料的記憶體
- **`async/await`：** 等 API 回應才繼續往下執行；有 `await` 的函式必須宣告 `async`
- **單字母變數慣例：** `o`（object）/ `s`（單筆項目）/ `i`（index）/ `e`（event）/ `res`（response）；往上找 `.map()` 就知道是什麼
- **`const` 的本質：** 宣告不可重新賦值的變數，存字串/數字不需要箭頭，存函式才有 `=>`
- **箭頭函式：** ES6 加入的語法，`const fn = (param) => ...` 等同 `function fn(param) { ... }`
- **參數型別定義位置：** 直接寫在括號內 `(orderId: number)`，因為參數是呼叫時才傳入的值，不能在外面定死
- **`...prev`（展開運算子）：** 下次繼續，從 `setShipments(prev => ({ ...prev, [orderId]: res.data }))` 開始
