### 跨域 Cookie 問題修正

#### 問題根因
- 前端（Vercel）與後端（Render）域名不同
- 手機瀏覽器把跨域 Cookie 當作第三方 Cookie 擋掉
- 導致登入後所有 API 請求回傳 401

#### 解法：Vercel Proxy（方案B）
- 新增 `vercel.json`：`/api/*` 請求 rewrite 到 Render
- `.env.production` 改為空字串（`VITE_API_URL=`）
- 前後端變成同域，Cookie 正常傳送

#### AuthController.java
- Cookie 改用 `ResponseCookie`
- 加入 `SameSite=None; Secure` 屬性（為跨域準備）

#### 環境變數修正
- `authApi.ts` / `InventoryPanel.tsx` / `ScanPanel.tsx` / `TransactionHistory.tsx` / `DiscountPanel.tsx`
- `|| 'http://localhost:8080'` 全部改為 `?? ''`
- 原因：`||` 把空字串視為 falsy，導致 fallback 到 localhost

## 踩到的坑

### `|| ''` vs `?? ''`
**症狀：** `.env.production` 設為空字串，但 Vercel build 後仍打 localhost。

**原因：** `||` 運算子把空字串視為 falsy，fallback 到 `'http://localhost:8080'`。

**解法：** 改用 `??`（Nullish Coalescing），只有 `null` / `undefined` 才 fallback。

### Vercel build 快取
**症狀：** push 後 bundle 名稱沒有改變，代表 Vercel 用了舊的 build。

**原因：** 檔案內容沒有實際改變時，Vite 的 hash 不會變。

**解法：** Redeploy without cache，或確認檔案內容確實改變。

### Safari 跨域 Cookie
**症狀：** Chrome 電腦版正常，iPhone Safari / Chrome 無法登入後呼叫 API。

**原因：** Safari 對第三方 Cookie 限制更嚴，`SameSite=None` 也可能被擋。

**解法：** Vercel Proxy 讓前後端同域，Cookie 變成第一方，Safari 也能正常運作。
