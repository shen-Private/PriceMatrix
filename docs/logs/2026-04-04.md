## Header 導航分隔線 ✅

### 修改內容
- `mainLinks` 陣列加入 `{ label: '---' }` 作為分隔符標記
- 渲染時判斷 `link.label === '---'` → 輸出 `<span>` 分隔線
- `visibleMain` filter 改為 `!l.action || can(l.action)`，讓分隔符不被過濾掉

### 分組結果
```
客戶管理 潛在客戶 | 折扣管理 報價單 訂單管理 | 改善提案 | 庫存總覽
```

### 學習概念
- `filter` 是 JS 原生陣列方法，不是 React 專屬
- `n => n > 2` 是箭頭函式的簡寫，等同 `function(n) { return n > 2; }`
- `const` = 容器規則（這個名字綁死了）；`new` = 物件的來源（根據 class 製造）

## Improvement 模組 ✅

### 背景
改善前公司的提案系統缺陷：
- 只評價短期效應，中長期無法追蹤
- 提案只能填一個人名字，無法記錄多人協作

### DB（5 張表）
```sql
improvement_proposal   -- 提案主體
improvement_member     -- 協作成員（多人）
improvement_milestone  -- 執行階段
improvement_report     -- 頻測報告
improvement_attachment -- 附件連結
```

### 後端
- `entity/`：5 個 Entity，`@OneToMany` / `@ManyToOne` 互相關聯，`@JsonIgnoreProperties` 防止無限遞迴
- `repository/`：5 個 Repository，含 `findByStatusOrderByCreatedAtDesc` / `findByProposalIdOrderByOrderIndex` 等查詢
- `service/`：`ImprovementService`，含提案 CRUD、狀態流轉、milestone 完成自動檢查、頻測報告新增
- `controller/`：`ImprovementController`，`/api/improvements` 下所有端點，`request.getAttribute("username")` 取得登入帳號

### 狀態流轉
```
DRAFT → PENDING_REVIEW → ACTIVE → COMPLETED
                       ↘ REJECTED
DRAFT / ACTIVE → CANCELLED
```

### 前端（ImprovementPanel.tsx）
- 提案列表 + 狀態篩選 Tab
- 展開行：提案內容 / 執行階段 兩個 Tab
- 新增/編輯提案 Modal（含協作成員 tag 輸入）
- 審核 Modal（核准 / 拒絕 + 意見）
- Milestone 新增 + 標記完成
- 頻測報告新增 + 附件連結

### 路由
- `App.tsx`：`<Route path="/improvements" element={<ImprovementPanel />} />`
- `Header.tsx`：`mainLinks` 加入「改善提案」

### 設計決策
- 每月提交數量限制 → 不寫進系統，由公司管理規則決定
- 資料整理（封存等）→ 等使用方需求再調整
- 審核權限 → 目前所有人可見，之後再加 `can()` 篩選
- 通知功能 → Backlog
