## 釐清的設計決策

### 委外報價邏輯

#### 委外直送
```
廠商報價（含運費）+ 公司利潤（固定值，業務可改）= 客人單價
```

#### 委外經倉
```
廠商報價 + 我方倉庫→客人運費 + 公司利潤 = 客人單價
```

#### 運費門檻（共通）
- 合計未滿一萬円 → 運費客人負擔（報價單上顯示運費）
- 合計一萬円以上 → 運費我方負擔（顯示「送料無料」或不顯示）

#### 設計方針
- 公司利潤預設固定值，業務可手動覆蓋
- 系統自動帶入預設值，業務判斷情況不同時自行修改
- 「業務自行判斷」的部分系統不強制計算，給可覆蓋欄位

- 新舊系統同時跑，新系統缺的資料去舊系統查，順手補進新系統
