## ✅ 完成的功能

### 1. HelloController - 基础 API
```java
@RestController
public class HelloController {
    @GetMapping("/hello")
    public String sayHello() {
        return "Hello PriceMatrix!";
    }
}
```
**测试：** `http://localhost:8080/hello`  
**结果：** `Hello PriceMatrix!`

---

### 2. 带参数的 API
```java
@GetMapping("/greet")
public String greet(@RequestParam String name) {
    return "Hello, " + name + "!";
}
```
**测试：** `http://localhost:8080/greet?name=shen`  
**结果：** `Hello, shen!`

**遇到的问题：**
- 参数名称不匹配错误
- 学到：`@RequestParam` 的参数名要和 URL 查询字串一致

---

### 3. 回传 JSON 物件

**Customer 类别：**
```java
public class Customer {
    private Long id;
    private String name;
    private String email;
    
    public Customer(Long id, String name, String email) {
        this.id = id;
        this.name = name;
        this.email = email;
    }
    
    // Getter 方法...
}
```

**API 方法：**
```java
@GetMapping("/customer")
public Customer getCustomer() {
    return new Customer(1L, "张三", "zhangsan@example.com");
}
```
**测试：** `http://localhost:8080/customer`  
**结果：** `{"id":1,"name":"张三","email":"zhangsan@example.com"}`

**关键学习：** Spring Boot 自动将 Java 物件转换成 JSON

---

### 4. 带参数查询 JSON
```java
@GetMapping("/customer/search")
public Customer searchCustomer(@RequestParam Long id) {
    if (id == 1) {
        return new Customer(1L, "张三", "zhangsan@example.com");
    } else if (id == 2) {
        return new Customer(2L, "李四", "lisi@example.com");
    } else {
        return new Customer(id, "未知客户", "unknown@example.com");
    }
}
```
**测试：**
- `?id=1` → 张三
- `?id=2` → 李四  
- `?id=999` → 未知客户

## 📚 核心概念理解

### 1. Spring Boot 请求处理流程
```
浏览器发送请求
  ↓
Tomcat 服务器接收
  ↓
DispatcherServlet (Spring Boot 核心调度员)
  ↓
HandlerMapping (路由表) 找到对应方法
  ↓
执行 Controller 方法
  ↓
HttpMessageConverter 转换回传值
  ↓
送回浏览器
```

---

### 2. 关键注解 (Annotation)

**@RestController**
- 声明这是 REST API 控制器
- 让 Spring Boot 自动扫描并注册
- 自动将回传值转换成 JSON

**@GetMapping("/path")**
- 处理 GET 请求
- 指定 URL 路径
- 简化版的 `@RequestMapping`

**@RequestParam**
- 接收 URL 查询字串参数
- 格式：`?key=value`
- 参数名要匹配

---

### 3. URL 查询字串结构
```
http://localhost:8080/greet?name=shen&age=25
└────┬────┘└────┬─────┘└┬─┘└─┬──┘└────┬─────┘
   协议      主机      端口  路径   查询字串

查询字串格式：
- ? 开始
- key=value
- & 连接多个参数
```

## 🧠 声明式 vs 命令式编程

**理解度：4-6 分（有概念但不确定）**

### WordPress 方式（命令式）
```php
add_action('rest_api_init', function() {
    register_rest_route('myplugin/v1', '/hello', [
        'methods'  => 'GET',
        'callback' => 'my_callback'
    ]);
});
```
→ 主动告诉框架「怎么做」

### Spring Boot 方式（声明式）
```java
@RestController
public class HelloController {
    @GetMapping("/hello")
    public String sayHello() {
        return "Hello!";
    }
}
```
→ 只描述「是什么」，框架自动处理

**目前理解：**
- ✅ 知道 Annotation 是「标签」
- ✅ 知道 Spring Boot 会自动扫描
- ⚠️ 对「为什么这样设计」还不太确定
- ⚠️ 需要更多实作来深化理解

## 💡 重要发现

### 1. Import 很重要
```java
import org.springframework.web.bind.annotation.RequestParam;
```
缺少 import 会编译错误

### 2. 参数名称要匹配
```java
@RequestParam String name  →  ?name=xxx  ✅
@RequestParam String shen  →  ?name=xxx  ❌
```

### 3. Spring Boot 的自动转换很强大
```java
return new Customer(...);  
→ 自动变成 JSON！
```
