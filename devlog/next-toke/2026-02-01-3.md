## 🧠 核心概念已理解

### 1. Java vs JDK vs Spring Boot

**Java:**
- 程式语言规范
- 定义语法规则

**JDK:**
- Java 开发工具包
- 包含编译器、执行器、标准库

**Spring Boot:**
- 网页应用框架
- 简化 Java 网页开发

**类比关系:**
```
PHP 语言        →  Java 语言
PHP 安装包      →  JDK
WordPress       →  Spring Boot
```

---

### 2. 编译型 vs 解释型

**编译型 (Java):**
```
写 code → 编译成 .class → 执行
优点: 执行快
缺点: 改了要重新编译
```

**解释型 (PHP):**
```
写 code → 直接执行
优点: 改了马上生效
缺点: 执行较慢
```

**Hot Reload 最佳平衡:**
```
Java + Spring Boot DevTools
= 编译型的效能 + 解释型的方便性
```

---

### 3. JIT 编译器预热

**机制:**
```
程式启动 → 解释模式执行 → 监控热点
→ 重新编译热点 → 存到 Code Cache
→ 之后执行直接用优化版本 → 越跑越快!
```

**效果:**
```
第 1 次: 500ms
第 100 次: 100ms
```

**记忆体占用:**
- Code Cache: ~240MB
- 有上限,不会无限增长

---

### 4. Maven 依赖管理

**pom.xml 作用:**
- 定义项目资讯
- 管理依赖
- 配置编译流程

**操作流程:**
1. 编辑 pom.xml
2. 点击 IntelliJ 的 "m" 图标 (Reload)
3. Maven 自动下载依赖
