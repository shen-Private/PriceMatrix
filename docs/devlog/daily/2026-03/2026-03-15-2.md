## 踩到的坑

### Vercel build 失敗
**症狀：** `Error: No Output Directory named "build" found`

**原因：** Vite 的 output 目錄是 `dist`，不是 CRA 的 `build`。

**解法：** Vercel dashboard → Build & Development Settings → Output Directory 改為 `dist`。

### createItemWithProduct 方法貼到 class 外面
**症狀：** 編譯錯誤。

**原因：** 複製貼上時把方法貼到 `}` 後面。

**解法：** 確認方法在 class 大括號內。

### ManufacturerRepository import 路徑錯誤
**症狀：** `cannot find symbol: ManufacturerRepository`

**原因：** `ManufacturerRepository` 在 `pricing` 模組，不在 `inventory`。

**解法：** import 改為 `com.pricematrix.pricematrix.pricing.repository.ManufacturerRepository`

### stockType ENUM 型別不符
**症狀：** `incompatible types: String cannot be converted to StockType`

**原因：** `CreateItemRequest` 的 `stockType` 宣告為 `String`。

**解法：** 改為 `InventoryItem.StockType`，加對應 import。

### 新增商品後盤點調整失敗
**症狀：** `Stock not found for item: 76`

**原因：** `createItemWithProduct` 原本沒有建立 `inventory_stock`。

**解法：** 新增商品時同步建立初始庫存（quantity = 0），僅限 `internal` / `outsource_warehouse`。
