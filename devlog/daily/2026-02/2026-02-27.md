## 學到的概念

### .then() 與 Promise

`axios.get()` 回傳的是 Promise（承諾之後會給答案），`.then()` 是「等拿到答案後執行這段」。

```jsx
axios.get('http://localhost:8080/api/categories')
  .then(response => {
    setCategories(response.data);  // response 是整個回應物件，.data 是資料本體
  });
```

**語法糖**：同樣功能、更好讀的寫法。`async/await` 是 `.then()` 的語法糖，結果完全一樣：

```jsx
const response = await axios.get(url);
setCategories(response.data);
```

### API 安全性：JWT Token 認證

API 預設是裸奔的，知道網址就能拿資料。正式做法是加 JWT Token：

1. 使用者登入 → 後端發給 Token（加密字串，內含身份與到期時間）
2. 之後每次請求 Header 帶上 Token
3. 後端驗證 Token → 有效才回傳資料，無效回傳 `401 Unauthorized`

Spring Boot 那側用 **Spring Security** 設定路徑權限：
```java
.requestMatchers("/api/auth/login").permitAll()  // 登入不需要 Token
.anyRequest().authenticated()                    // 其他都要驗證
```

Token 儲存位置：
- `localStorage`：簡單，但有 XSS 風險（惡意 JS 可讀取）
- `HttpOnly Cookie`：JS 完全讀不到，更安全，業界較推薦

### Cookie 是什麼

瀏覽器幫你存的一小包資料，每次發請求時自動帶上去。登入後伺服器發 Session Cookie，之後每次開頁面瀏覽器自動帶上，伺服器認出你就不用重新登入。

**XSS 攻擊**：惡意 JS 用 `document.cookie` 讀走 Cookie，拿著別人的 Cookie 直接以對方身份登入。`HttpOnly Cookie` 的防護就是讓 JS 根本讀不到。

### style 裡的條件渲染

用三元運算子在 style 裡做條件樣式：

```jsx
style={{ backgroundColor: isHover ? "red" : "blue" }}
// isHover 是 true → 紅，false → 藍

style={{ backgroundColor: isHover ? "red" : "" }}
// false 時不套樣式
```

複雜條件拉到 return 外面先算好：

```jsx
let bgColor;
if (isActive && isHover) {
  bgColor = "red";
} else if (isActive) {
  bgColor = "blue";
} else {
  bgColor = "gray";
}

return <button style={{ backgroundColor: bgColor }}></button>;
```

`else if` = 「否則如果」，處理三種以上情況。

### state 當開關：!（反轉）

```jsx
onClick={() => setIsActive(!isActive)}
// !isActive：true 變 false，false 變 true，每按一次切換
```

`setIsActive(false)` 永遠只會設成 false，沒有切換效果。

---

### const vs let

```js
const color = "red";
color = "blue";  // ❌ 不能重新賦值

let color = "red";
color = "blue";  // ✅ 可以重新賦值
```

定下來不會變 → `const`，需要被重新賦值 → `let`。

### Props：父元件傳資料給子元件

```jsx
// 子元件：接收 props
function MyButton({ label, color }) {
  return (
    <button style={{ backgroundColor: color }}>
      {label}
    </button>
  );
}

// 父元件：傳入不同值
function Practice() {
  return (
    <div>
      <MyButton label="確認" color="green" />
      <MyButton label="取消" color="red" />
    </div>
  );
}
```

概念：**共用的抽出來，變動的從外面給**。跟 ACF + template parts 邏輯一樣，但 props 可以傳資料、函數、物件，不只是文字。

**元件命名規則**：
- 小寫開頭 `<button>` → HTML 原生標籤
- 大寫開頭 `<MyButton>` → 自訂 React 元件（React 規定，用來區分）

### whileHover 不需要手動追蹤 hover state

```jsx
// ✅ Framer Motion 自己處理 hover
<motion.button
  whileHover={{ scale: 1.2, backgroundColor: "red" }}
>
```

不需要 `isHover` state，`whileHover` 本身就是「hover 時套用這些樣式」。
