## 學到的概念

### 三元運算子

```jsx
條件 ? 真的時候 : 假的時候
```

`?` 不是比較，是根據條件選一個值。`===` 才是比較符號。

### 賦值 vs 比較

```jsx
const isSelected = filterType === k;
//              ↑ 賦值(=)   ↑ 比較(===)
```

右邊先執行比較得到 true/false，再用 `=` 存進變數。

### 物件的動態取值 [k]

```jsx
STOCK_TYPE_COLOR[k]
// k = 'internal' → STOCK_TYPE_COLOR['internal'] → '#2ecc71'
```

用變數當 key 動態取物件的值。

### HEX 顏色透明度

```jsx
STOCK_TYPE_COLOR[k] + '22'
// '#2ecc71' + '22' → '#2ecc7122'
// 最後兩碼是透明度，22 ≈ 13% 不透明
```

### React 條件樣式 vs CSS 原生條件

- CSS `:is` / `:not` → 根據元素狀態，靜態，跟資料無關
- React 條件樣式 → 根據資料狀態，動態，跟變數連動
- WP 用 ID 指定樣式的做法在靜態內容可行，但 React 管的是動態資料，所以在 JS 層處理更直接

### CSS Modules

```tsx
import styles from './InventoryPanel.module.css';
className={styles.btnSearch}
```

自動加 hash 避免 class 撞名，component 級別管理，不像 WP 全域 CSS 要小心命名。

### 多個 class 的寫法

```jsx
className={`${styles.btnSearch} ${styles.btnIcon}`}
// 條件式
className={`${styles.btnSearch} ${isSelected ? styles.active : ''}`}
```

### JSX 規則

- `return` 只能回傳一個根元素
- 多個元素要用 `<div>` 或 `<>` 包起來
- `<style>` 不能直接在 JSX 裡寫 CSS

### inline style 的限制

```jsx
style={{ backgroundColor: 'red' }}  // 可以
style={{ hover: ... }}              // 不行，hover 只能用 CSS 檔案
```

### 事件監聽器

```jsx
onClick        // 點擊
onChange       // 值改變
onMouseEnter   // 滑鼠移入
onMouseLeave   // 滑鼠移出
```

### useState 完整理解

```jsx
const [isHover, setIsHover] = useState(false);
//     ↑ 當前值   ↑ 改變的方法        ↑ 初始值
```

- `isHover` → React 幫你記著的當前狀態
- `setIsHover` → 改變狀態的唯一方法，呼叫後畫面自動更新
- 不能直接 `isHover = true`，React 不會知道值變了

初始值：
```jsx
useState(false)  // 明確知道初始狀態
useState([])     // 一開始是空陣列（等 API 回來）
useState(null)   // 有東西但還不知道是什麼
useState()       // undefined，等同 false，但不建議
```

### hover 改變內容的完整寫法

```jsx
const [isHover, setIsHover] = useState(false);

<div
  onMouseEnter={() => setIsHover(true)}
  onMouseLeave={() => setIsHover(false)}
  style={{ backgroundColor: isHover ? 'red' : 'white' }}
>
  {isHover ? '滑鼠在上面' : '練習'}
</div>
```

## 架構理解

**Component 的樹狀結構**
```
App.tsx
└── Practice（頁面級，Route 指向這裡）
    └── Test（子 component，由 Practice 呼叫）
```

Route 只管「哪個 URL 對應哪個頁面級 component」，頁面內部的結構由 component 自己管。

**一個檔案一個 component** 是正常的管理方式，技術上可以多個但不建議。
