## 學到的概念

### 前端兩層架構

原本用原生 HTML/CSS/JS 設想的兩層，在 React 裡天然存在：

| | Local State | Server State |
|---|---|---|
| 資料從哪來 | component 自己 | 後端 API |
| 影響範圍 | 只影響自己 | 影響多個 component |
| 用什麼 | `useState` | `useState` + `useEffect` + `axios` |

**第一層（Local State）**：滑鼠 hover、按鈕樣式變化，用 `useState` 放在 component 內部。

**第二層（Server State）**：從後端拿資料、送資料給後端，用 `useEffect` + `axios`。

### 兩層的邊界

```jsx
// StockTypeButton 不知道點下去會發生什麼
// 它只負責「被點了就呼叫 onClick」
// 誰呼叫它，誰決定點下去做什麼
<StockTypeButton onClick={() => setFilterType(k)} />
```

### 檔案管理

一個 component 一個檔案是標準做法：

```
StockTypeButton.tsx   ← 第一層
InventorySidebar.tsx  ← 第二層，import StockTypeButton 進來用
```

### TypeScript 在 `.jsx` 裡不能用

`interface`、`useState<Type[]>` 等 TS 語法只能在 `.tsx` 檔案裡使用。
練習時可以先用 `.jsx` 跳過 TS，之後再補型別。

### % 取餘數（循環技巧）

```js
clickCount % COLORS.length
// clickCount 一直加，但結果永遠在 0 ~ (length-1) 之間循環
// 0%4=0, 1%4=1, 2%4=2, 3%4=3, 4%4=0 ← 回到起點
```

### .length

JS 內建屬性，回傳陣列或字串的長度：

```js
['red', 'blue', 'green'].length  // → 3
'hello'.length                   // → 5
```

### 陣列 vs 物件

```js
// 陣列：有順序，用數字取值
const arr = ['red', 'blue', 'green'];
arr[0]  // → 'red'

// 物件：無順序，用名字取值
const obj = { internal: '內部庫存' };
obj['internal']  // → '內部庫存'
obj.internal     // → 同上
```

### Object 常用方法

```js
const obj = { internal: '內部庫存', outsource_infinite: '外包無限' };

Object.keys(obj)
// → ['internal', 'outsource_infinite']   只拿 key

Object.values(obj)
// → ['內部庫存', '外包無限']              只拿 value

Object.entries(obj)
// → [['internal', '內部庫存'], ['outsource_infinite', '外包無限']]
// key 和 value 一起拿，之後同時需要兩者時用
```

### map 裡面輸出多個元素

```jsx
// map 每次只能回傳一個根元素
// 多個元素要用 <> 包起來
{Object.keys(STOCK_TYPE_LABEL).map(k => (
  <>
    <p key={k + '_1'}>{STOCK_TYPE_LABEL[k]}</p>
    <p key={k + '_2'}>{STOCK_TYPE_COLOR[k]}</p>
  </>
))}
// key 不能重複，所以加 _1 / _2 區分
```

### p 標籤不能巢狀

```jsx
// ❌ 錯誤：瀏覽器會靜默出錯，畫面全白
<p>
  {items.map(item => <p key={item.id}>{item.name}</p>)}
</p>

// ✅ 正確：外層改用 div
<div>
  {items.map(item => <p key={item.id}>{item.name}</p>)}
</div>
```
