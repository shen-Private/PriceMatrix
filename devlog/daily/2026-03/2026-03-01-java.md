## 一個請求的完整流向

```
瀏覽器 / axios
    ↓ HTTP 請求（GET /api/categories）
Controller        ← 從這裡開始讀
    ↓ 呼叫
Service           ← 業務邏輯
    ↓ 呼叫
Repository        ← 資料庫操作
    ↓ SQL
Database
    ↑ 回傳資料，原路返回
```

**閱讀方法：從 Controller 的 `@GetMapping` / `@PostMapping` 開始，Ctrl + 點擊方法名稱往下追。**

## 學到的概念

### import 語法

```java
import org.springframework.web.bind.annotation.*;
// .* = 把這個包裡所有東西全部匯入
// 不用一個一個寫 @RestController、@GetMapping...
```

### `final` 關鍵字

```java
private final CategoryService categoryService;
// 賦值後不能再換，等同前端的 const
```

### Java 變數宣告規則

Java 所有變數都必須先宣告型別：

```java
型別 變數名稱 = 值;

String sql = "SELECT * FROM category";
int count = 0;
Category c = new Category();
List<Category> list = new ArrayList<>();
```

---

### `this` 關鍵字

```java
public CategoryController(CategoryService categoryService) {
    this.categoryService = categoryService;
    // this.categoryService = 自己身上的變數
    // categoryService      = 外面傳進來的參數
}
```

`this` = 指這個 Class 自己本身的變數。

---

### 方法回傳型別

Java 要求事先聲明回傳型別：

```java
public List<Category> getAllCategorys() {
//     ↑回傳型別          ↑方法名稱
    return categoryRepository.findAll();
}
```

前端不用聲明，Java 必須寫。

---

### `List<Category>` 的意思

```java
List<Category>
// 容器<裡面裝什麼>
// = 裝著 Category 物件的清單
// 等同前端 TypeScript 的 Category[]
```

---

### `interface` 與 `extends`

```java
public interface CategoryRepository extends JpaRepository<Category, Long>
```

- `interface` = 只定義「能做什麼」，不寫怎麼做（像合約）
- `extends` = 繼承，把 JpaRepository 所有方法全部拿過來用

### JpaRepository vs 原生 JDBC

**原生 JDBC（公司舊系統會看到）：**

```java
// 每個方法都要寫這一大串
Connection connection = DriverManager.getConnection(url, user, password);
String sql = "SELECT * FROM category";
PreparedStatement stmt = connection.prepareStatement(sql);
ResultSet rs = stmt.executeQuery();

List<Category> list = new ArrayList<>();
while (rs.next()) {
    Category c = new Category();
    c.setId(rs.getLong("id"));
    c.setName(rs.getString("name"));
    list.add(c);
}
return list;
```

**JpaRepository（現代做法）：**

```java
// 一行搞定
return categoryRepository.findAll();
```

---

### PreparedStatement 與 SQL Injection

```java
// ❌ 危險：直接拼字串
String sql = "SELECT * FROM category WHERE id = " + id;
// 使用者輸入 "1 OR 1=1" → 撈出整張表
// 使用者輸入 "1; DROP TABLE category--" → 刪表

// ✅ 安全：用 ? 佔位
String sql = "SELECT * FROM category WHERE id = ?";
PreparedStatement stmt = connection.prepareStatement(sql);
stmt.setLong(1, id);  // 值被當純文字處理，不會被執行
```

---

### ResultSet

資料庫回傳的原始結果，不是物件清單，必須用 `while` 一行行讀：

```java
while (rs.next()) {   // 有下一行 → true，沒有 → false，跳出
    rs.getLong("id");
    rs.getString("name");
}
// 轉成 List 之後才能用 forEach
```

---

### Java 的 forEach vs 前端

```java
// Java
list.forEach(item -> {
    System.out.println(item.getName());
});

// 前端 JS
list.forEach(item => {
    console.log(item.getName());
});
```

差別只有 `=>` 換成 `->`。

### 數字型別

```java
int    // 最大約 21億，一般數字
Long   // 最大約 900京，id 通常用這個
double // 有小數點，價格之類的
```

id 用 Long 是慣例，怕資料筆數太多 int 不夠用。

### Entity 解析

```java
@Entity                               // 這個 Class 對應一張資料庫表
@Table(name = "pricing_categories")   // 對應的表名

@Id                                   // 主鍵
@GeneratedValue(strategy = GenerationType.IDENTITY)  // id 自動遞增

private Long id;
private String name;

// getter / setter：讓外部讀寫私有變數
public Long getId() { return id; }
public void setId(Long id) { this.id = id; }
```

JpaRepository 透過 Entity 上的 `@Table` 知道要去哪張表操作。
