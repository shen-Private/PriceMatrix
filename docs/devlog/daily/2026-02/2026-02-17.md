踩過的坑
坑一：儲存中提示沒有出現
原因：editingId 還沒清除，畫面還在顯示輸入框，<span> 裡的「儲存中...」根本沒機會渲染。
解法：setSavingId 之後立刻 setEditingId(null)，讓畫面先切回顯示模式。
坑二：新增失敗 500 錯誤
原因：資料庫有 UNIQUE(customer_id, product_id) 限制，同客戶同商品不能有兩筆折扣。
解法：不是 bug，是正確防護。測試時換沒有折扣的商品即可。

## 今日學到的概念

### MySQL vs H2
H2 是記憶體資料庫，Spring Boot 關掉資料就消失。
MySQL 是真正的資料庫，資料永久保存。
application.properties 就是告訴 Spring Boot「去哪裡找資料庫」的設定檔。

### MySQL Workbench
視覺化管理 MySQL 的工具。
可以直接下 SQL 查詢確認資料是否正確寫入。

### Port 3306
MySQL 預設監聽的埠號。
Spring Boot 透過 jdbc:mysql://localhost:3306/pricematrix 連線。
