## 踩過的坑

**tsconfig.json 預設值衝突**
`npx tsc --init` 產生的設定含 `verbatimModuleSyntax: true`，與 CRA 的 webpack 環境不相容，導致所有 import/export 報錯。換成 CRA 官方建議的設定解決。

**moduleResolution 需改為 bundler**
Storybook 新版型別定義用較新的模組解析格式，`moduleResolution: "node"` 找不到。改為 `"bundler"` 後解決。

**CSS Modules 找不到型別宣告**
TS 預設不認識 `.module.css`，需要在 `src/declarations.d.ts` 加上宣告才能 import。

**Storybook import 路徑錯誤**
`@storybook/react` 是 Renderer 層，新版 Storybook ESLint 規則要求從 Framework 層 import。改為 `@storybook/react-webpack5` 解決。

**檔案內容混入**
改名過程中不小心把 stories 內容貼進 `DiscountPanel.tsx`，導致 `Identifier 'DiscountPanel' has already been declared` 錯誤。手動清除多餘內容解決。

## 學到的概念

**ES Module vs CommonJS**
- CommonJS：`require` / `module.exports`（Node.js 傳統格式）
- ES Module：`import` / `export`（瀏覽器原生現代格式）
- `verbatimModuleSyntax` 要求純 ESM 環境，CRA 是混用環境所以衝突

**型別 import vs 值 import**
```ts
import axios from 'axios';                          // 值 import — 執行時存在
import type { Meta, StoryObj } from '@storybook/react-webpack5'; // 型別 import — 編譯後消失
```

**Create React App（CRA）是什麼**
React 是函式庫，CRA 是工具鏈（用 webpack 打包、啟動開發伺服器）。CRA 是 2016 年 Meta 官方出的，現在幾乎停止維護。業界主流已換成 Vite。

**Storybook 兩層架構**
- Renderer 層（`@storybook/react`）：負責渲染 React 元件
- Framework 層（`@storybook/react-webpack5`）：負責打包工具整合，內部已包含 Renderer
