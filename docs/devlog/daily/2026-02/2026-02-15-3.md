## 踩過的坑

### 坑一：transient instance
前端送商品名稱，JPA 不知道對應哪筆資料庫記錄。
解法：Service 先用名稱查出有 id 的 Product 物件，再存 Discount。

### 坑二：createdAt is null
`nullable = false` 但新增時沒給值，資料庫拒絕寫入。
解法：欄位宣告時給預設值 `= LocalDateTime.now()`

### 坑三：useState 放錯位置
useState 被放進 handleDelete 函數裡面，React 規定 Hook 只能放在元件最上層。
解法：移到 function DiscountPanel() 的最上面區塊。

## 今日學到的概念

### Service 層的職責
使用者只填「人需要決定的東西」（商品名稱、折扣率）
Service 負責補齊「系統自動知道的東西」（時間、關聯物件）
資料庫只收完整正確的資料

### React Hook 規則
useState 只能放在元件函數的最上層
不能放在 if、for、或其他函數裡面
