隨手筆記



快照就是「抄一份存起來」，之後不管原本的資料怎麼變，你的那份不動。







前端 → CORS 檢查 → JWT 驗證 → Controller







Controller: 接收 HTTP → 回传 JSON

Service:    业务逻辑（这次还很简单）

Repository: 资料存取（SQL 操作）

Database:   储存资料



什么是 Hibernate？

定义：

Hibernate = ORM 框架

ORM = Object-Relational Mapping（物件关联映射）



JSX = HTML + JavaScript 混合

Hibernate 負責翻譯：



你寫 item.getProduct() → 它自動翻成 SELECT \* FROM pricing\_products WHERE id = ?

你寫 quoteRepository.save(quote) → 它自動翻成 INSERT INTO sales\_quote ...









創新端點的順序：由內而外

Entity → Repository → Service → Controller



查錯誤的順序：由外而內

錯誤訊息 → Controller → Service → Repository → Entity → DB

一個判斷原則

錯誤分兩種：



編譯錯誤（cannot find symbol、TS2307）→ 一定是 import、變數名、路徑寫錯，直接找那個檔案改

執行錯誤（500、NullPointerException）→ 邏輯問題，看 IntelliJ 的 console log，會告訴你哪一行出錯



資料的流向是：

DB → Repository撈出來 → 變成Entity → Service處理 → Controller回傳 → Jackson轉JSON → 前端



簡單記：陣列用數字當 key，物件用名字當 key。



{} → 切換模式 / 包住多行

() → 呼叫函數 / 包住 return

\[] → 陣列
